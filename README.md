# tdd
writes Test-Driven Development

1. テストを書く
2. コンパイラを通す
3. テストを走らせ、失敗を確認する
4. テストを通す
5. 重複を排除する

十分な量のテストがない場合は、あるべきテストを描いてからリファクタリングを行う。
